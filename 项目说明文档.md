# 项目结构与功能说明文档

本文档详细说明了工程中各个文件夹及核心文件的功能。

## 📁 文件夹功能说明

| 文件夹名称 | 功能描述 |
| :--- | :--- |
| **media** | **媒体文件存储**：存放用户上传的头像、检测过程中的临时文件、以及处理后的结果图片和视频。 |
| **models** | **模型存放目录**：存放自己训练好的最优模型文件（如 `best.pt`），系统默认调用此目录下的模型。 |
| **newsite** | **Django核心目录**：包含项目的核心配置、后端逻辑代码、路由配置等。 |
| **pretrained** | **预训练模型文件夹**：存放官方或基础的YOLO预训练模型（如 `yolov8n.pt`, `yolo11n.pt`），用于迁移学习。 |
| **runs** | **训练结果文件夹**：包含模型训练过程中生成的各类数据，如训练日志、权重文件（weights）、性能指标图表等。 |
| **static** | **静态资源文件夹**：存放网页前端所需的CSS样式表、JavaScript脚本、字体文件及静态图片资源。 |
| **templates** | **前端页面文件夹**：存放系统的所有HTML页面模板（如登录页 `login.html`、主页 `user_main.html` 等）。 |
| **test_media** | **测试素材文件夹**：存放用于测试检测功能的图片和视频资源。 |
| **train_data** | **训练数据集**：存放用于模型训练的图像数据及其对应的标签文件（包含训练集和验证集）。 |

## 📄 主要文件功能说明

| 文件名称 | 功能描述 |
| :--- | :--- |
| **config_text.py** | **文本配置文件**：集中管理界面显示的文字（如系统标题、欢迎语）以及类别名称的中英文映射（`CLASS_NAME_MAP`）。 |
| **db.sqlite3** | **数据库文件**：SQLite数据库，存储用户信息、历史检测记录、知识库等数据。 |
| **manage.py** | **Django管理脚本**：Django项目的入口脚本，用于启动Web服务器、执行数据库迁移、创建管理员等操作。 |
| **requirements.txt** | **依赖环境文件**：记录了项目运行所需的所有Python库及其版本，用于在新环境中快速部署。 |
| **yolo_detection.py** | **核心算法代码**：封装了YOLO模型的加载（含缓存机制）、推理检测、画图及中文字符绘制逻辑。 |
| **newsite/views.py** | **视图逻辑代码**：Django后端的核心业务逻辑，处理用户登录、文件上传、实时视频流处理、检测请求等。 |
| **newsite/settings.py** | **项目配置文件**：Django项目的全局配置，包含数据库设置、已安装应用、静态文件路径、时区设置等。 |
| **newsite/models.py** | **数据模型文件**：定义了数据库的表结构，例如用户表 (`CustomUser`)、检测记录表 (`FlowerDetectionRecord`)。 |
| **newsite/urls.py** | **路由配置文件**：定义了URL地址与后端视图函数的对应关系，决定了访问某个网址时由哪个函数处理。 |
| **run_train.py** | **训练启动脚本**：用于独立启动模型训练任务的脚本，包含训练参数的配置。 |
| **run_detect_video.py** | **视频检测脚本**：用于独立运行视频文件检测的脚本（非Web方式）。 |
